### 生命周期 Bat 用法

**Online.bat** - 上线时调用（确认 CLI 已启动）
**Done.bat** - 任务完成时调用
**Blocked.bat** - 遇到阻塞时调用
**Failed.bat** - 任务失败时调用
**{CUSTOM_STAGE_BATS}** - 池自定义阶段 BAT，由用户补充

调用格式（Claude 在 bash 中运行，请用下面这种写法）：
```
cmd //c ".\\Online.bat $AGENT_ID $TASK_ID {POOL_NAME} online"
cmd //c ".\\Done.bat $AGENT_ID $TASK_ID {POOL_NAME} done"
```

### 执行流程（模板）

1. 读取当前目录下的任务文件（task_*.txt，已由 Runtime 搬运到此）
2. 理解任务指令，提取 TASK_ID
3. 立即调用 Online.bat 确认已上线
4. 按本池专属流程执行任务
5. 所有最终交付物先写入 `workspace/`
6. 任务完成后调用 Done.bat
7. 调用 Done.bat 后立即显式退出或结束对话

### 强制规则

- 禁止以 end_turn 或空回复结束对话
- 调用终态 bat 后直接结束，不要继续对话
- 禁止调用任何 Skill
- 任何需要保留的最终产物都必须先写入 workspace/
- 环境变量 AGENT_ID、TASK_ID、POOL、SIGNAL_SERVER_PORT 已由 Runtime 设置
